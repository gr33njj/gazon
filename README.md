# GAZON — GameProject

Симулятор работы в ПВЗ (пункт выдачи заказов), чёрная комедия, от первого лица.
Портируется из HTML/Three.js MVP (`../GAZON PVZ Simulator.dc.html`) в Unity 6, целевая платформа — PC/Steam.

## Что уже есть в этой папке
- `Docs/GDD.md` — механики, извлечённые из MVP построчным разбором кода.
- `Docs/Architecture.md` — план AI-first экосистемы разработки (MCP-серверы, веб-панель) по фазам,
  с обоснованием порядка.
- `Docs/Decisions.md` — журнал решений.
- `Design/` — черновики по миру, экономике, клиентам (дополняет GDD, не дублирует).
- `AI/` — правила и контекст для Claude Code (или любого другого агента) в этом репозитории.
- `Database/schema.sql` + `Database/project.sqlite` — база игрового контента (архетипы клиентов,
  реплики, штрафы, мемы, апгрейды донат-магазина, товары бабушки, звания), пересобирается через
  `python3 Tools/ai_suite/init_db.py`.
- `.claude/CLAUDE.md` — инструкции для Claude Code.

## Чего здесь нет и почему
Нет `Assets/`, `Packages/`, `ProjectSettings/` — то есть самого Unity-проекта. Эта VPS (`gr33njj.dev`)
не имеет установленного Unity Editor и не подходит для его запуска (см. `Docs/Architecture.md`,
раздел «Ограничения текущей машины»). Сама разработка в Unity идёт на ваших локальных машинах.

## Следующие шаги (сделать вручную, не с этой машины)

1. Один из вас ставит **Unity Hub + Unity 6** и создаёт новый проект по шаблону **3D (URP)**
   с именем `GAZON` (или как договоритесь).
2. Содержимое этой папки (`Docs/`, `Design/`, `AI/`, `Database/`, `Tools/`, `.claude/`) копируется
   внутрь корня созданного Unity-проекта, рядом с `Assets/`.
3. Создаётся GitHub-репозиторий, в него коммитится результат шага 2 плюс `.gitignore` (уже есть здесь).
4. Начинается порт вертикального среза: докстанция → полки → окно выдачи → таймер смены, без
   мини-игр и телефона — просто чтобы убедиться, что цикл ощущается так же, как в вебе.
5. Только после того как вертикальный срез играбелен — переходить к MCP-серверам и веб-панели
   по фазам из `Docs/Architecture.md`. Не раньше: без работающего Unity-прототипа непонятно,
   какие из инструментов реально нужны в первую очередь.

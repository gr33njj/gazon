# Tools/ai_suite

Здесь будут жить Python MCP-серверы (Unity bridge, project_db, doc_search, git_guard) —
см. `AI/MCP.md` и `Docs/Architecture.md` (фаза 2). Пока в этой папке только один рабочий скрипт:

- **`init_db.py`** — пересобирает `Database/project.sqlite` из `Database/schema.sql` и сид-данных,
  извлечённых из HTML MVP (реплики клиентов, штрафы, мемы, апгрейды донат-магазина, товары бабушки,
  звания). Запуск: `python3 Tools/ai_suite/init_db.py`. Идемпотентный — можно гонять сколько угодно раз,
  каждый прогон пересоздаёт файл с нуля из этого скрипта, а не апдейтит существующие ряды.

Когда контент начнёт меняться не только через сид (т.е. кто-то руками правит записи через SQL/будущий
Project MCP), `init_db.py` перестанет быть источником истины по данным — тогда сид нужно будет вынести
в отдельный экспорт из живой БД, а не наоборот. Отметить это решение в `Docs/Decisions.md`, когда до
этого дойдёт.
